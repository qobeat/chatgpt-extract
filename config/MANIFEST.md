# config/ — MANIFEST

Hand-curated configuration: the model bank, pricing, plans, and build defaults.
All committed files here MUST be free of personal paths (enforced by
`tests/test_repo_hygiene.py`).

## How an agent EXECUTES this folder
- Not executed; loaded at runtime by `lib/paths.py` (build config) and
  `lib/models_bank.py` (model bank). `gpt summarize` with no model prints the
  merged bank.
- Personal/local settings live ONLY in gitignored siblings:
  `reconstruct.config.local.json` (e.g. `data_root`, `default_zips`) and
  `models.local.json`. Never put personal data in the committed files.

## How an agent CHANGES this folder
- Each JSON validates against a schema in `schema/`; keep them in sync and re-run
  `pytest -q` (`test_schema_validation.py`). Bump the relevant schema + a
  migration note if you change a shape.
- Add models to `models.json` (provider + billing only). Machine-generated
  benchmark verdicts do NOT go here — they live in `generated/` and are merged at
  load time. Subscription billing references `plans.json` by `plan_id`.

## Files
- `models.json` — model → provider + runtime options + billing (validated by
  `schema/models_bank.schema.json`).
- `pricing.json` — token prices (validated by `schema/pricing.schema.json`).
- `plans.json` — subscription plans referenced by `billing.plan_id`
  (`schema/plans.schema.json`).
- `reconstruct.config.json` — committed build defaults (no personal paths).
- `reconstruct.config.example.json` — template for the gitignored local config.
- `generated/model_benchmarks.json` — machine-generated benchmark verdicts
  (regenerated by `scripts/gen_model_benchmarks.py`; merged into the bank at load).
