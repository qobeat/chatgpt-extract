# docs/ — MANIFEST

Generated reports, dated benchmark write-ups, build logs, and design notes.
Reference material — not executed, not a governance authority.

## How an agent EXECUTES this folder
- Not executed. `cross-sweep-report.md` is regenerated by `./gpt report`; the
  benchmark/validation docs are dated records of specific sweeps.

## How an agent CHANGES this folder
- `cross-sweep-report.md` is OUTPUT — regenerate it via `gpt report` rather than
  hand-editing. Dated docs (e.g. `benchmark-20260626.md`) are historical; prefer
  adding a new dated doc over rewriting an old one.
- Build logs record what a change session did; append a new one rather than
  rewriting history. Authority for requirements/version lives in
  `REQUIREMENTS.md` / `CHANGELOG.md`, not here.

## Files (representative)
- `cross-sweep-report.md` — generated unified cross-sweep model report.
- `benchmark-20260626.md`, `benchmark-oct2024.md`, `timing-oct2024-posthoc.md`,
  `validation-smoke-20260624.md` — dated benchmark/validation records.
- `build-log-*.md` — per-session build logs (e.g. semantic-ask, ADOS geometry).
- `REDESIGN-PROPOSAL.md`, `PLAN-ADDENDUM-models-json.md` — design notes.
- `README-legacy.md`, `AI_MODEL_TESTS-legacy.md` — retained legacy docs.
